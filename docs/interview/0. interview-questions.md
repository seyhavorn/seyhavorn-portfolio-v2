### Technical Questions

**Q: What is the difference between `==` and `===` in JavaScript?**

**A:** "`==` is loose equality - it compares values after type coercion. `===` is strict equality - it compares both value AND type without coercion.

```javascript
5 == '5'   // true (coerces string to number)
5 === '5'  // false (different types)
```

I always use `===` to avoid unexpected bugs from type coercion."

---

**Q: Explain OOP concepts: inheritance, encapsulation, polymorphism.**

**A:** 
- **Inheritance:** A class can inherit properties and methods from a parent class. Example: `class Dog extends Animal`.
- **Encapsulation:** Bundling data and methods together, hiding internal state. Example: private variables with public getters/setters.
- **Polymorphism:** Objects can take multiple forms. A child class can override parent methods. Example: `Animal.speak()` vs `Dog.speak()`.

These principles make code more modular, reusable, and maintainable.

**Q: What is a database index? How does it improve performance?**

**A:** "An index is a data structure that speeds up data retrieval. It works like a book's index - instead of scanning every page (full table scan), you look up the page number directly.

For example, indexing a `user_id` column lets the database find records instantly using B-tree lookup instead of scanning millions of rows.

Trade-off: Indexes speed up reads but slow down writes (INSERT/UPDATE) because the index must be updated too."

**Q: What is the difference between GET and POST in HTTP?**

**A:**
| GET | POST |
|-----|------|
| Data in URL query string | Data in request body |
| Visible in browser history | Not visible in history |
| Limited data size | Can send large payloads |
| Idempotent (safe to retry) | Not idempotent |
| Used for fetching data | Used for creating/updating |

Example: Searching users = GET. Creating a user = POST.

**Q: Explain JavaScript closures and promises.**

**A:** 

**Closure:** A function that remembers its outer scope even after the outer function returns.

```javascript
function createCounter() {
  let count = 0;
  return function() {
    count++;
    return count;
  };
}
const counter = createCounter();
counter(); // 1
counter(); // 2
```

**Promise:** An object representing async operation completion.

```javascript
fetch('/api/users')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error(error));
```

Or with async/await:
```javascript
async function getUsers() {
  const response = await fetch('/api/users');
  const data = await response.json();
  return data;
}
```

**Q: What is CORS? How do you fix CORS errors?**

**A:** "CORS (Cross-Origin Resource Sharing) is a security feature that blocks web pages from making requests to a different domain than the one that served the page.

Error example: `Access to fetch at 'https://api.example.com' from origin 'https://myapp.com' has been blocked by CORS policy.`

Fixes:
1. **Backend:** Add CORS headers to allow specific origins:
   ```javascript
   // Express.js
   app.use(cors({ origin: 'https://myapp.com' }));
   ```
2. **Proxy:** Use a proxy server in development
3. **Server config:** Configure Nginx/Apache to add `Access-Control-Allow-Origin` header

Never disable CORS in production - it's a security feature!"

## 🔥 More Technical Q&A (Advanced)

### Database Questions

**Q: What is a transaction? Explain ACID properties.**

**A:** "A transaction is a sequence of database operations that are treated as a single unit. Either all succeed or all fail.

**ACID properties:**
- **Atomicity:** All operations complete or none do (all-or-nothing)
- **Consistency:** Database remains in valid state before and after
- **Isolation:** Concurrent transactions don't interfere with each other
- **Durability:** Once committed, changes persist even after crash

Example: Bank transfer - debit from Account A AND credit to Account B must both succeed or both fail."

---

**Q: What is the N+1 query problem? How do you fix it?**

**A:** "N+1 happens when you execute 1 query to fetch parent records, then N queries to fetch related child records (one per parent).

```javascript
// BAD: N+1 problem
const users = await User.findAll(); // 1 query
users.forEach(user => {
  user.posts = await Post.findAll({ where: { userId: user.id } }); // N queries
});

// GOOD: Eager loading
const users = await User.findAll({ include: [Post] }); // 2 queries total
```

Fix: Use eager loading (`include` in Sequelize, `with()` in Laravel, `JOIN` in SQL)."

---

**Q: Explain database normalization forms.**

**A:** 
- **1NF:** Each column contains atomic values (no arrays/lists in cells)
- **2NF:** Meets 1NF + no partial dependencies (all non-key columns depend on entire primary key)
- **3NF:** Meets 2NF + no transitive dependencies (non-key columns don't depend on other non-key columns)

Example of violation:
```
Orders(order_id, customer_id, customer_name, product_id, product_price)
```
`customer_name` depends on `customer_id`, not `order_id` → violates 3NF.

Fix: Split into separate tables (Orders, Customers, Products)."

**Q: How do you prevent SQL injection?**

**A:** "SQL injection happens when user input is directly concatenated into SQL queries.

```javascript
// VULNERABLE ❌
const query = `SELECT * FROM users WHERE email = '${email}'`;

// SAFE ✓ (Parameterized queries)
const query = 'SELECT * FROM users WHERE email = ?';
db.execute(query, [email]);

// SAFE ✓ (ORM/Query Builder)
await User.findOne({ where: { email } });
```

Always use parameterized queries, prepared statements, or ORM methods. Never concatenate user input!"

**Q: What is caching? When should you use it?**

**A:** "Caching stores frequently accessed data in fast storage (memory) to reduce database load and improve response time.

Use caching when:
- Data is read often but rarely changes (user profiles, settings)
- Expensive computations (reports, aggregations)
- External API calls with rate limits

Don't cache:
- Frequently changing data (stock prices, real-time inventory)
- User-specific sensitive data without proper invalidation

Tools: Redis, Memcached, browser cache, CDN cache."

### Frontend Questions

**Q: What is the difference between `let`, `const`, and `var`?**

**A:**
| Keyword | Scope | Hoisting | Reassignment | Temporal Dead Zone |
|---------|-------|----------|--------------|-------------------|
| `var` | Function | Yes | Yes | No |
| `let` | Block | Yes | Yes | Yes |
| `const` | Block | Yes | No | Yes |

```javascript
var x = 1;       // Function-scoped, can redeclare
let y = 2;       // Block-scoped, can reassign
const z = 3;     // Block-scoped, cannot reassign

if (true) {
  var x = 10;    // Affects outer scope!
  let y = 20;    // New variable, scoped to if block
}
```

Best practice: Use `const` by default, `let` when reassignment needed, avoid `var`."

---

**Q: Explain event delegation in JavaScript.**

**A:** "Event delegation attaches one event listener to a parent element instead of multiple listeners to children. The event bubbles up from target to parent.

```javascript
// BAD: Listener on every button
document.querySelectorAll('.item').forEach(item => {
  item.addEventListener('click', handleClick);
});

// GOOD: One listener on parent
document.getElementById('list').addEventListener('click', (e) => {
  if (e.target.matches('.item')) {
    handleClick(e);
  }
});
```

Benefits: Better performance, works for dynamically added elements, less memory usage."

---

**Q: What is the difference between `null` and `undefined`?**

**A:**
- **`undefined`:** Variable declared but not assigned a value
- **`null`:** Explicitly assigned to represent "no value" or "empty"

```javascript
let name;           // undefined (not assigned)
let age = null;     // null (explicitly empty)

console.log(name);  // undefined
console.log(age);   // null
```

Type checking:
```javascript
typeof undefined  // "undefined"
typeof null       // "object" (JavaScript quirk!)
```

Best practice: Use `null` when intentionally clearing a value."

**Q: What is load balancing? Why is it important?**

**A:** "Load balancing distributes incoming traffic across multiple servers to prevent overload and ensure high availability.

Benefits:
- **Scalability:** Handle more users by adding servers
- **Reliability:** If one server fails, others take over
- **Performance:** Users connect to nearest/fastest server

Algorithms:
- **Round Robin:** Distribute evenly in rotation
- **Least Connections:** Send to server with fewest active connections
- **IP Hash:** Same user always goes to same server (sticky sessions)

Tools: Nginx, HAProxy, AWS ELB, Cloudflare."

**Result:** Query time dropped from 8s to 120ms. Dashboard now loads instantly."
