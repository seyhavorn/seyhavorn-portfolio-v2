SMS Java Service — Multi-Tenant SMS Gateway Microservice
Backend Developer
Java 21 · Spring Boot 3.2.3 · Spring WebFlux · PostgreSQL 17 · Redis · RabbitMQ 3.10 · Resilience4j · ShedLock · Docker · Jib
Bronx Technology

- Developed a provider-agnostic SMS gateway microservice with Command Pattern architecture and per-client JWT authentication, enabling multiple tenants to send SMS and OTP verifications through pluggable providers (PlasGate, NomSa) with zero code changes for new integrations.

- Implemented complete OTP lifecycle management (generation, delivery, JWT phone token issuance, verification with attempt tracking), multi-tenant client isolation with independent JWT secrets, and a Redis-based credit/subscription system for high-throughput scenarios.

- Built production-grade resilience with Resilience4j circuit breakers for Redis/RabbitMQ, ShedLock distributed scheduling locks, exponential backoff retry policies, and WebClient connection pooling (500 max connections).

- Deployed on AWS EC2 ARM64 (Graviton) using Google Jib for daemonless Docker builds, a 3-step scripted pipeline (Build → Save → Deploy), Docker Compose orchestration, and Springdoc OpenAPI (Swagger UI) for API documentation.
